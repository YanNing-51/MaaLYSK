## 概念介绍

- **Task** | **任务**

  MaaLYSK最左侧的任务列表。如`✈️恋与深空启动`、`🎮周常娃娃机`就是一个个任务。

- **Config** | **配置/实例**

  MaaLYSK页面最上方的一横排，可以自定义每一个任务选项、模拟器连接，并且保存。

- **Preset** | **预设**

  增加实例时，可以选择内置的预设，如`日常任务`、`周常任务`等。

主页面所有按钮在鼠标悬停后，都会有浮窗进行按钮的介绍。不知道按钮操作的，可以悬停鼠标，或参考[快速开始-软件页面](/zh_cn/manual/1.1.html#软件界面)介绍。

- 右键每一个配置，可对配置进行操作。

## 开机启动（win）

MaaLYSK没有自带的开机启动功能，但可以通过Win自带功能实现开机启动。

如有问题，自行上网搜索`windows实现软件开机启动`。

1. 打开MaaLYSK文件夹，找到`MaaLYSK.exe`，右键`创建快捷方式`。

## 多开操作

MaaLYSK可以对添加的每一个实例单独进行设备连接、设置路径等操作。

### 准备工作

1. 进行模拟器多开：复制模拟器，或新增一个模拟器并下载恋与深空。

   - MuMu：通过多开管理器复制已有模拟器（需先关闭），或新增安卓12模拟器并下载游戏。
   - Bluestacks：通过多开管理器创建新实例，编辑参数。
   - 雷电：通过多开管理器复制模拟器**连接配置**：默认自动检测；手动时需下载platform tools并配置ADB路径，通过 `adb devices` 测试连接。

2. 获取模拟器ADB端口：打开模拟器设置，寻找ADB端口相关内容，记下对应模拟器的端口号。

   > [!TIP]
   >
   > - 由于每个模拟器设置页面不同，具体操作上网查找或询问模拟器客服！
   > - 常见模拟器：
   >   - MuMu 12：多开器右上角查看；
   >   - MuMu 5.0：模拟器设置-其他-ADB端口；
   >   - Bluestacks：模拟器设置→高级→Android调试（ADB）。
   >
   > - 通用方法：
   >   - 方案1（ADB命令）：启动模拟器后，在ADB所在目录执行`adb devices`（Windows CMD/PowerShell），输出中的 `127.0.0.1:<端口>` 或 `emulator-<四位数字>` 即为连接地址。
   >   - 方案2（资源监视器）：Win+S搜“资源监视器”→网络选项卡→监听端口中找模拟器进程（如HD-Player.exe）的端口，或TCP连接中找adb.exe对应的远程端口。

3. 添加配置：MaaLYSK上方`+`号`添加实例`，或右键`复制标签页`，勾选设置好对应任务项。

### 设备连接

1. 先打开模拟器，再打开MaaLYSK进行对应设备连接

- 自动连接
  1. 在对应配置页点击连接设备的`刷新`按钮，寻找模拟器
  2. 点击下拉 `▾`按钮，找到对应模拟器`端口号`并选择。

- 手动填写：点击`自定义`按钮，手动选择`ADB路径`并填写`ADB地址`。参照[常见问题-手动连接模拟器](/zh_cn/manual/2.2.html#手动连接模拟器)
  - ADB路径：`模拟器安装路径/xx/xx/adb.exe`
  - ADB地址：`127.0.0.1:<端口号>`/`emulator-<端口数字>`

<div align="center">
  <img src="/image/tutorial_image/advanced/多开操作.png" alt="多开操作"/>
</div>

2. 设置好对应配置后，在`设置`-`连接设置`中，勾选`记住连接`，记住对应连接。根据自己需求，开关`连接设置`页面的按钮。

::: info 注

同时有多个不同模拟器（MuMu+雷电）的，保证先开要连MaaLYSK的那个模拟器占住端口，才能让MaaLYSK连接对应模拟器。

:::

## 外部通知

在`设置`-`外部通知`中进行设置，支持任务完成后发送“任务已全部完成”或自定义内容的通知。

### 钉钉

电脑端钉钉建群并添加自定义机器人：群设置 → 机器人 → 添加机器人 → 自定义 → 添加。

安全设置选“加签”，复制记录密钥和 webhook 地址。

打开`设置` - `外部通知` - `开启钉钉通知`，将密钥和地址分别粘贴至对应位置。

- **机器人 Token** ：webhook 地址（例如 `https://oapi.dingtalk.com/robot/send?access_token=xxx`）
- **钉钉 Secret** ：密钥

### 邮箱

登录邮箱网页端，进入设置 → 账号与安全 → POP3/IMAP/SMTP/Exchange/CardDAV 服务，开启服务并生成授权码（不同邮箱步骤类似）。

复制授权码，粘贴至MaaLYSK指定位置，点击“发送测试”接收验证通知。

- **机器人 Token** ：你的 QQ 邮箱账号（如 `xxx@qq.com`）
- **邮箱 Secret**：生成的授权码（不是登录密码）

### 飞书（Lark）

方式 A：Webhook（极简，推荐）

- **Webhook URL**：直接填飞书机器人 Webhook

方式 B：签名模式（普通）

- **Lark ID**：机器人 ID
- **Bot Token**：机器人密钥

### WxPusher

方式 A：普通推送

- **Bot Token**：AppToken
- **UID**：用户 UID

方式 B：极简推送

- **Bot Token**：留空
- **UID**：填写 SPT

### Telegram

- **Chat ID**：聊天或群组 ID
- **Bot Token**：机器人 Token

### Discord Bot

- **Channel ID**：频道 ID
- **Bot Token**：机器人 Token

### Discord Webhook

- **Webhook URL**：Webhook 链接
- **Webhook Name**：显示名称（可留空）

### OneBot

- **Server**：OneBot 服务地址（如 `http://127.0.0.1:5700`）
- **Key**：鉴权 Key（若服务需要）
- **User**：接收消息的 QQ 号

> [!TIP] 说明
>
> - 该渠道目前未接入统一发送流程，需要自行接入后才能使用。

### SMTP

- **SMTP Server**：服务器地址（如 `smtp.qq.com`）
- **SMTP Port**：端口（常见 465/587/25）
- **SMTP SSL**：SSL 开关
- **SMTP Auth**：是否需要账号密码
- **SMTP User**：账号（仅在 Auth 开启时填写）
- **SMTP Password**：密码/授权码（仅在 Auth 开启时填写）
- **SMTP From**：发件人地址
- **SMTP To**：收件人地址

### Qmsg

- **Server**：Qmsg 服务地址
- **Key**：Qmsg Key
- **User**：接收者 QQ
- **Bot**：机器人 QQ（可留空）

### Server酱

- **Send Key**：Server 酱 SendKey（支持 `sctp` 形式）

### 通用 Webhook

- **Webhook URL**：完整 `http/https` 地址
- **Content Type**：默认 `application/json`
- **Payload Template**：模板中使用 `{message}` 作为占位符

> **示例模板**
>
> ```bash
> {"text":"{message}"}
> ```
